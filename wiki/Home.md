# Project Wiki

Welcome to the dev-machine-setup wiki — human-and-agent-readable documentation for this codebase. The sections below follow the Diátaxis convention ([ADR 0004](https://github.com/alexherrero/agentic-harness/blob/main/wiki/architecture/decisions/0004-diataxis-documentation-spec.md)): each page is written for a single reader intent — doing or understanding — and never mixes modes.

> [!NOTE]
> This page is maintained by the `documenter` sub-agent at phase boundaries. Humans may edit it directly; the sub-agent preserves human edits.

## 🔧 Trying to do something specific?

How-to guides are task-focused recipes for users who already know the basics.

- [Bootstrap a new Mac](Bootstrap-A-New-Mac) — fresh Mac → full GUI + CLI dev environment in one command.
- [Bootstrap a new Debian or Ubuntu host](Bootstrap-A-New-Debian-Or-Ubuntu) — fresh Linux host → CLI-only dev environment.

## 💡 Want to know why?

Explanation pages capture intent, rationale, and trade-offs — the *why*, not the *how*.

- [Dev-machine setup — design](Dev-Machine-Setup-Design) — intent, shape, and trade-offs of the OS-dispatch architecture.
- [Architecture decisions](Decisions) — ADRs for non-obvious choices.
